## What's Checked vs What's Missing

CHECKED:
- arg1_3 ~= arg2: Player identity matches
- arg2_2 ~= "SkillName": Skill name matches expected

NOT CHECKED:
- Skill ownership (player has unlocked skill)
- Player state (alive, not stunned/frozen)
- Server-side cooldown
- Required weapon equipped
- Resource availability
- Class/role requirements
- Skill prerequisites

## Impact

- Use any skill regardless of unlock status
- Use skills while in disabled states (stunned, frozen, dead)
- Use skills without required weapons
- Use skills from other weapon classes
- Bypass resource/energy costs
- Use premium/paid skills without purchasing

## Recommended Fix Location

Implement validation in the shared Skills module or create
a middleware function that all skill modules call before
executing their logic.

Example fix pattern:
```lua
local validation = Skills.validateSkillExecution(arg2, arg1, "SkillName")
if not validation.success then
    warn("Skill validation failed:", validation.reason)
    return
end
-- Proceed with skill execution
```
